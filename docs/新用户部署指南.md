# 新用户部署指南

> 本文档写给**你（主库管理员）**：如何把房屋管家分发给新用户，让他在自己的电脑和手机上都能使用，且两端数据互通。

---

## 核心原理

```
新用户桌面端 ──┐
               ├──► 你的服务器 (8.166.122.38:9091) ──► SQLite 数据库
新用户手机端 ──┘
```

- **服务端是唯一的数据源**，所有客户端通过网络连接同一台服务器。
- 桌面端和手机端读写的是同一份数据，**天然互通，无需额外配置**。
- 新用户自己注册账号，数据与你（主库管理员）的数据互相隔离。

---

## 前提条件（你需要确保的）

| 条件 | 说明 |
|------|------|
| 服务器持续在线 | `8.166.122.38:9091` 保持运行，否则所有客户端都不可用 |
| 防火墙开放 9091 端口 | 云服务器安全组需要放行 TCP 9091 |
| 新用户知道服务器地址 | 告诉新用户：`http://8.166.122.38:9091` |
| GmAersMess 仅限你的电脑 | 管理员账号绑定了你电脑的 MachineGuid，新用户无法使用（安全设计） |

## 一、打包桌面客户端（你的操作，只做一次）

### 命令解释

```bash
pyinstaller --onedir --windowed --name "楼房管理系统" --clean house_management.py
```

| 参数 | 含义 |
|------|------|
| `pyinstaller` | PyInstaller 打包工具 |
| `--onedir` | 输出一个**文件夹**（exe + 库文件），比单个 exe 更稳定 |
| `--windowed` | 不弹出黑色命令行窗口，只显示 GUI |
| `--name "楼房管理系统"` | 程序名称 |
| `--clean` | 打包前清理临时文件 |
| `house_management.py` | 要打包的 Python 源文件 |

### 操作步骤

1. 确保已安装 PyInstaller：
   ```bash
   pip install pyinstaller
   ```

2. 在项目根目录执行打包命令：
   ```bash
   pyinstaller --onedir --windowed --name "楼房管理系统" --clean house_management.py
   ```

3. 打包完成后，`dist/楼房管理系统/` 文件夹结构：
   ```
   楼房管理系统/
   ├── 楼房管理系统.exe    ← 主程序，双击运行
   ├── python3.dll
   ├── _tkinter.pyd
   ├── tcl/               ← tkinter 界面依赖
   └── ... （约 20-30 个库文件）
   ```

4. **右键** `楼房管理系统` 文件夹 → 「发送到」→「压缩(zipped)文件夹」

5. 把 `楼房管理系统.zip` 发给新用户。

### 也可用快捷方式（以后更新时）

直接双击项目根目录的 `build.bat`，它内部执行的就是同样的 PyInstaller 命令。

## 二、新用户桌面端操作（新用户要做的）

1. **解压** `楼房管理系统.zip` 到任意文件夹（桌面、文档等都可）。

2. **双击** `楼房管理系统.exe`，弹出登录窗口。

3. 登录窗口顶部有「服务器地址」输入框，默认显示 `http://localhost:9091`：
   ```
   ┌──────────────────────────────────┐
   │  服务器地址                       │
   │  ┌──────────────────────────────┐│
   │  │ http://localhost:9091        ││  ← 改成 http://8.166.122.38:9091
   │  └──────────────────────────────┘│
   └──────────────────────────────────┘
   ```

4. 填入用户名和密码，点击「注册」创建新账号。

5. 注册成功后自动登录，进入主界面即可使用。

6. 以后每次打开程序，服务器地址和账号会自动记住，无需重新输入。

> ⚠️ 桌面上首次运行后，`client_config.json` 会自动生成在 exe 同目录下，**不要删除它**（里面保存了登录凭证和服务器地址）。

---

## 三、手机端操作（新用户要做的）

1. 安装你发给他的 APK 文件。

2. 打开 App，进入登录界面。

3. 点击「服务器设置」展开服务器地址输入框。

4. 填入 `http://8.166.122.38:9091`。

5. 用**和桌面端相同的账号**登录。

6. 登录后即可看到与桌面端相同的数据——**互通完成** ✅。

## 四、常见问题

### Q1：新用户桌面端打不开，报错或闪退？
- 检查 exe 所在路径**不要有中文或空格**。
- 尝试以管理员身份运行。
- 如果还不行，让新用户安装 Python 3.8+，直接用源码运行（见下方 Q3）。

### Q2：新用户连不上服务器？
- 确认你的服务器 `8.166.122.38:9091` 正在运行。
- 确认云服务器防火墙/安全组已开放 TCP 9091。
- 让新用户在浏览器访问 `http://8.166.122.38:9091/api/v1/health`，应返回 `{"status":"ok"}`。

### Q3：不用 exe，新用户能用源码直接运行吗？
可以。新用户需要：
1. 安装 Python 3.8+（勾选 "Add Python to PATH"）
2. 把 `house_management.py` 和 `api_client.py` 放到同一文件夹
3. 命令行执行：`python house_management.py`
4. 登录界面上修改服务器地址即可

### Q4：新用户可以注册多个账号吗？
可以，系统支持多用户。登录界面提供「注册」和「登录」两种模式，切换即可。

### Q5：新用户的数据和我（主库管理员）的数据会混在一起吗？
不会。每个用户只能看到自己创建的楼房，以及别人授权给他看的楼房。数据是隔离的。
